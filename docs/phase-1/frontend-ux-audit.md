# DropRoom 前端产品与 UX 审查

## 审查结论

经产品范围复核，一期采用单房间界面：首页只负责创建或加入，房间页只展示当前房间。多房间列表和快速切换推迟到二期。本轮同时修复小屏滚动、Dark 模式、邀请链接复制和房间内昵称同步问题。

## 本轮已修复

- 首页保留明确的创建和加入入口，不展示房间列表。
- 房间页桌面端展示成员侧栏，移动端通过抽屉查看成员。
- 移动端房间顶栏使用展开侧栏图标打开成员抽屉，抽屉内使用对应的收起图标关闭。
- 成员栏上下分隔线统一使用主题边框色，暗色模式下不再出现突兀的亮白线。
- 创建房间改为一键完成，系统自动生成默认名称，房主进入后仍可修改。
- 移除创建和加入流程中的人为延迟，操作完成后立即反馈并进入目标房间。
- 房主同时拥有“退出”和“解散”入口，避免把普通离开误导为销毁房间。
- 退出、解散和删除文件改为明确说明影响范围的确认弹窗。
- 邀请链接和文字复制增加失败反馈。
- 已销毁或不存在的房间只提供返回首页入口，不再误导用户重新加入。
- 邀请链接加入页复用首页顶栏，新成员可以先修改昵称，再确认加入房间。
- 邀请链接加入卡片在手机窄屏下保留左右安全留白，不与屏幕边缘贴合。
- 加入、改名和昵称输入增加可见标签、回车提交、长度限制和空值校验。
- 文字上限修正为20,000字符；单批文件修正为最多50个、总计500,000,000字节。
- 房间容量修正为2,000,000,000字节并按十进制单位展示；顶栏使用占用进度图，上传预占计入占用量，右侧展示“已占用 / 总容量”。
- 房间倒计时与容量区统一使用线性进度轨道和等宽数字，临近到期时继续使用黄、红两级提醒。
- 桌面端房间顶栏的退出和更多操作使用一致的轻量按钮样式。
- 移动端房间顶栏展示真实房间名称，不再使用含义较弱的“当前房间”。
- 首页在低高度竖屏和手机横屏下允许纵向滚动，不再裁切操作区与生命周期提示。
- 首页、编辑弹窗和错误页统一使用语义化主题颜色，支持浅色、深色和跟随系统。
- 房间内设置昵称会同步当前房间成员信息，不再只修改本地偏好。
- 传输气泡始终贴近头像；悬浮操作栏显示在气泡外侧，自己的内容位于左侧，对方的内容位于右侧；短文字仍保留便于悬浮和复制的最小操作宽度。
- 文字发送期间显示加载状态并阻止按钮或回车重复提交，失败后保留原输入。
- 上传卡片区分排队、上传、暂停、失败、取消和删除状态；删除后的文件保留不可用提示。
- 小文件显示连续总体进度，多分片文件额外显示当前分片进度；空文件在前端直接拒绝并提示。
- 浏览器基础样式统一使用 UnoCSS reset，并清理未使用的 Vite 模板说明与素材。
- 超过400行的前端文件已按房间同步、操作和弹窗职责拆分。

## API 接入结果

- Web 使用 Hono `hc` 直接消费后端 `AppType`，请求参数和响应类型不再手写。
- 创建、加入、快照、文字、成员、房主管理和文件操作均使用真实 HTTP API。
- 实时成员、房主转移、房间更新、内容和上传进度通过 SSE 同步。
- 本地仅保存昵称、房间恢复所需的轻量信息和偏好，不保存成员凭证，也不把本地快照作为房间内容的事实来源。
- 文件先由服务端批量预占容量，再使用 `p-limit` 控制最多3个并发分片请求。
- 二进制上传地址继续由 Hono `hc` 生成，发送使用浏览器原生上传事件补充分片内进度，不引入额外请求库。
- 文字输入框支持直接粘贴剪贴板图片和文件，并复用现有批次校验与上传流程。
- 上传中断后保留服务端偏移；当前页面可以直接继续，刷新后重新选择同一个文件可以恢复。
- 图片预览、下载和 SSE 使用 HttpOnly 房间 Cookie，URL 不再包含成员凭证。
- 房间码区域用于复制完整邀请链接，可以直接粘贴到微信或 QQ。

## 仍需验收

- 在目标桌面端和移动端浏览器验证大文件上传、断网恢复和 EventSource 自动重连。
- 生产镜像已固定使用同源 `/api`，部署时只需按生产域名填写后端 CORS 来源。
