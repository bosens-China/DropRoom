# DropRoom 浏览器兼容性与验收

## 支持范围

一期使用 Vite 8 的 Baseline Widely Available 生产目标，并在构建配置中固定最低版本，避免工具升级后支持范围自动变化。

| 平台                   |     最低版本 | 一期要求                             |
| ---------------------- | -----------: | ------------------------------------ |
| Chrome 桌面版          |          111 | 完整支持                             |
| Edge 桌面版            |          111 | 完整支持                             |
| Firefox 桌面版         |          114 | 完整支持                             |
| Safari macOS           |         16.4 | 完整支持                             |
| Safari iOS / iPadOS    |         16.4 | 完整支持                             |
| Android System WebView |          111 | 尽力支持，需要宿主允许文件选择和下载 |
| 其他 Chromium 浏览器   | Chromium 111 | 尽力支持                             |

低于最低版本时不保证页面能够加载、实时事件能够重连或大文件能够完整上传。项目不引入 legacy 构建和旧浏览器 polyfill。

## 验收环境

每次正式发布至少覆盖：

- 最新稳定版 Chrome、Edge、Firefox 和 Safari。
- 最低支持版本或对应的浏览器测试环境。
- 一台 iPhone 或 iPad 真机。
- 一台 Android 真机。
- 桌面宽屏、桌面窄窗口和移动端竖屏。

## 核心流程

每个目标浏览器需要通过：

1. 创建房间，复制邀请链接并在另一设备加入。
2. 刷新页面后使用 HttpOnly 房间 Cookie 恢复当前房间。
3. 发送、复制并重新加载文字历史。
4. 选择、粘贴、拖放并上传多个文件；粘贴纯文字仍进入输入框。
5. 中断上传后直接继续；刷新后重新选择同一个文件恢复上传。
6. 下载普通文件；预览并下载图片。
7. 验证300 MB单批、50个文件和1 GB房间容量提示。
8. 暂时断网后恢复网络，确认 EventSource 自动重连并继续收到事件。
9. 验证房主刷新、主动退出、解散和断线转移。
10. 验证30分钟提醒、5分钟强提醒和最终到期销毁。
11. 在设置中开启浏览器通知，验证权限申请、后台收到其他成员文字或文件时的通知，以及关闭后不再通知。
12. 在房间页分别验证 Windows/Linux `Ctrl+A` 与 macOS `Command+A` 选择文字输入框内容，同时不干扰其他输入框的原生全选。

## 移动端专项

- 从系统相册和文件选择器选择文件。
- 在低高度竖屏和手机横屏中确认首页所有入口与提示均可滚动访问。
- 聚焦发送框、昵称、房间名和加入房间等输入控件时页面不得自动放大；用户主动双指缩放仍然可用。
- 上传期间切到后台再返回，界面不得错误显示上传成功。
- 锁屏、切换 Wi-Fi 和移动网络后恢复页面。
- 下载行为交给浏览器或系统分享面板，不承诺固定保存目录。
- iOS 内存不足导致页面重载时，允许上传失败，但必须明确展示失败结果。

## 大文件与弱网

- 分别验证100 MB和接近300 MB的单批上传。
- 在上传中途断网，确认服务端保留已完成分片和容量预占。
- 网络恢复后点击“继续”，确认从已完成位置恢复。
- 刷新页面后重新选择同一个文件，确认从服务端记录的位置恢复。
- 选择不同文件时必须拒绝续传，不能合并成损坏文件。
