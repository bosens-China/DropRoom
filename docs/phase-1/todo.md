## 当前核对结论（2026-07-16）

- 前后端主体能力已经完成，创建与加入限流已接入 Redis。
- Web 已通过 Hono `hc` 接入真实 HTTP API，通过原生 EventSource 接入 SSE，并使用服务端临时存储完成文件上传下载。
- Web 一期采用单房间界面，首页负责创建或加入，房间页只展示当前房间；多房间管理推迟到二期。
- 前端产品与 UX 偏差、已修复项和后续后端接入项见 [前端产品与 UX 审查](./frontend-ux-audit.md)。
- 当前 Web lint、TypeScript 校验、54个测试和生产构建均通过。
- 当前 API lint、TypeScript 校验、构建和后端37个测试均通过，另有1个按环境跳过。
- 后端生产代码已按职责拆分，所有 TypeScript 文件均不超过400行。
- 已提供 API 与 Web 生产镜像；API 通过 `tsx` 启动源码，Web 由 Nginx 托管 Vite
  产物并将 `/api` 代理到后端。
- Vite 已按主要依赖拆包，Nginx 已配置哈希资源长期缓存和 gzip。
- 已补充生产 Compose 示例和双镜像 CI 发布流程。
- 成员上限采用“在线成员与5分钟重连保留成员合计9名”的口径。
- 文件上传已改为固定分片续传，刷新后重新选择同一个文件可以恢复任务。
- SSE、图片预览和下载使用每个房间独立的 HttpOnly Cookie，不再在 URL 中携带成员凭证。
- 浏览器支持矩阵、生产部署、告警阈值和异常清理手册已经补充；真实设备与真实域名验收仍需在部署环境完成。
- 自动化测试已验证 Cookie 鉴权、成员凭证不进入 URL，以及旧本地凭证缓存迁移。
- 上传卡片按创建时间固定位置；跨平台全选、已销毁房间刷新、两小时倒计时阈值及解散时下载中资源保留规则已完成自动化覆盖。
- 长纯文字编辑后在发送时自动转 TXT、按序命名、TXT 预览和文字气泡内部滚动已完成自动化覆盖。
