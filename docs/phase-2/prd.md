# DropRoom 二期产品需求文档

## 1. 文档信息

- 阶段：二期（房间管理增强）
- 状态：范围已确定
- 前置条件：一期完成真实浏览器和生产环境验收

## 2. 产品定位

DropRoom 是一个多人房间型的 Web 文件传输助手。它提供类似微信文件传输助手的体验，但允许多个设备或成员进入同一个临时房间。

二期继续服务短时多人传输，不扩展为聊天产品、社交产品或永久网盘。

## 3. 二期目标

二期只解决两个问题：

1. 房主可以移除不应继续留在房间的成员。
2. 用户可以清楚了解临时存储和数据删除边界。

## 4. 产品边界

- 用户仍然无需注册或登录。
- 房间仍然在连续5分钟无人在线时销毁。
- 房间创建满24小时后仍然强制销毁。
- 服务进程重启仍然视为全部临时房间销毁。
- 二期不增加聊天互动、跨房间关系或长期存储。

## 5. 功能需求

### 5.1 房主移除成员

- 房主可以从在线成员列表中移除除自己之外的成员。
- 移除操作需要确认，并明确说明该成员会立即离开房间。
- 成员被移除后，其当前成员凭证立即失效。
- 被移除成员返回加入房间流程。
- 移除成员不建立跨房间封禁关系。
- 房主退出、断线和转移规则继续遵循一期定义。

### 5.2 临时存储与隐私说明

- 页面明确说明文件由个人服务器临时保存。
- 页面明确说明连续5分钟无人在线后会删除整个房间。
- 页面明确说明房间创建满24小时后会强制删除。
- 页面明确说明服务进程重启也会提前销毁临时房间。
- 页面明确说明 DropRoom 不提供永久备份能力。
- 页面明确说明 DropRoom 不检查或验证用户传输的文件内容。
- 产品说明必须与实际存储、清理和日志行为保持一致。

## 6. 关键异常场景

- 房主移除成员时，成员已经离线或主动退出。
- 被移除成员继续使用旧凭证发起 HTTP 请求。
- 被移除成员继续使用旧凭证建立 SSE 连接。
- 被移除成员使用仍然有效的房间邀请重新加入。

所有异常都需要给出明确结果，不能让用户误以为成员仍然拥有原有房间权限。

## 7. 二期验收标准

1. 房主可以移除在线成员。
2. 被移除成员的旧凭证立即失效。
3. 被移除成员的 HTTP 请求和 SSE 重连都会被拒绝。
4. 移除成员不影响其他成员和房间内容。
5. 页面准确说明空房清理、24小时上限和服务重启后的删除规则。
6. 二期能力不改变一期的容量限制、匿名方式和短时中转定位。
